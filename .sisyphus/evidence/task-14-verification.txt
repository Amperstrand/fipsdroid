### Build Status
- ❌ Cargo build fails due to tun/netlink incompatibility on macOS
  - Issue: `tun = { version = "0.8.5", features = ["async"] }` pulls in `netlink-sys`
  - Root cause: `netlink-sys` uses Linux-specific constants (PF_NETLINK, SOCK_CLOEXEC, etc.)
  - These don't exist in macOS's `libc` (uses PF_HYLINK, O_CLOEXEC instead)
  - Compilation error count: 41 errors in netlink-sys crate

### Resolution Strategy
1. Replace `tun` with `tun2` (macOS-compatible alternative)
2. Update `rtnetlink` to use `netlink-packet-route` with platform guards
3. Or implement custom TUN device on macOS using `tun2` crate
